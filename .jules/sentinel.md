## 2024-05-12 - [XSS vulnerability via innerHTML]\n**Vulnerability:** XSS via innerHTML\n**Learning:** The application updates UI metrics including DOM elements directly using innerHTML, which is an anti-pattern. While the inputs in this case don't seem to be completely user-controlled (they are interval calculations and constants), setting innerHTML could be unsafe. Particularly, we should use safer DOM manipulation methods like element.textContent or creating elements securely instead of injecting raw HTML.\n**Prevention:** Avoid assigning to innerHTML. Use textContent instead or create DOM nodes specifically.
